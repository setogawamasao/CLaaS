# 要件分析

## 1. コンセプト：「人（大学生）をダメにするサービス」

> Larry Wall はプログラマーの三大美徳の1つとして **「怠慢（Laziness）」** を挙げました。
> CLaaS はその精神を大学生活に適用します。

**CLaaS（Campus Life as a Service）** は、"大学生活をAIによって最適化しすぎた結果、人をダメにしてしまう"ことをコンセプトにした、生成AI時代の大学生向け生活最適化OSです。

**「ぼっちでも、コネなしでも、大学生活を全力で楽をして楽しめる。」**

従来、大学生活を充実させるには「先輩とのつながり」が必要でした。楽単情報も、過去問も、麻雀や合コンの誘いも、就活情報も——すべては人脈がある人だけの特権でした。

**しかしCLaaSでは、それらをAIとプラットフォームが代わりにやります。**

- **①楽単時間割自動生成**: シラバスをAIが解析し「出席重要度」「レポート負荷」「テスト依存率」を算出。最小努力で最大単位を取れる時間割を自動生成します。
- **②マッチング機能**: 空きコマが重複するユーザーと麻雀・合コン・ゲームで自動マッチング。代返・教材フリマで学生間のリソースが流通します。
- **③空きコマ娯楽レコメンド**: 楽単情報によって生まれた空きコマに、「上映開始直前の映画」「この時間で読み終わる漫画」「就活情報」をレコメンド。
- **④ガクチカクレジット**: 怠慢な行動に対してクレジットが発行され、学生間で"大学生活リソース"が流通するトークンエコノミーを実現します。
- **⑤企業連携機能**: 映画館・出版社・マイナビなどの企業とも連携し、空きコマにぴったりのクーポンや就活情報が自動で届きます。

単位取得に必要な最低限の努力だけを残し、それ以外の時間を **"最も快適な暇つぶし"** へ最適化します。

**「大学生活のすべてを、サービス化する。」** それが、Campus Life as a Service です。

## 2. 課題分析

### ぼっち大学生が抱える「あるある」

#### 📚 学習面（コネがないと詰む）
| ぼっちあるある | CLaaSの解決策 | 対応機能 |
|---|---|---|
| 楽単情報を教えてくれる先輩がいない | 全大学生の知恵をアプリで共有・検索 | ① |
| 過去問を持っている友達がいない | 大学・授業別に過去問をアップロード・ダウンロード | ① |
| シラバスを読んでも難易度がわからない | AIが「出席重要度」「レポート負荷」「テスト依存率」を自動分析 | ① |
| 時間割を相談できる人がいない | AIが最小努力の時間割を3パターン自動生成 | ① |
| 使い終わった教科書を譲れる後輩がいない | 教材フリマで大学全体とつながる | ② |

#### ⏰ 時間面（空きコマが虚無）
| ぼっちあるある | CLaaSの解決策 | 対応機能 |
|---|---|---|
| 空きコマに誘ってくれる友達がいない | 同じ空きコマの人と麻雀・合コン・ゲームで自動マッチング | ② |
| 空きコマに何をすればいいか考えるのが面倒 | 残り時間に合わせたコンテンツを自動提案 | ③ |
| 近くの映画館・施設を調べるのが面倒 | 地図で自動表示 | ③ |
| 読み終わらない本を選んでしまう | 空きコマで読み終わる作品だけを提案 | ③ |

#### 👥 コミュニティ面（人脈ゼロからのスタート）
| ぼっちあるある | CLaaSの解決策 | 対応機能 |
|---|---|---|
| 麻雀・合コン・ゲームに誘ってくれる人がいない | 趣味コミュニティ・空きコママッチングで自動つなぎ | ② |
| 就活情報を教えてくれる先輩がいない | 企業連携で空きコマにぴったりの就活情報が届く | ⑤ |
| 授業に出席するのが面倒 | 代返機能（Trust_Scoreで信頼性管理） | ② |
| 情報共有しても自分に何のメリットもない | 投稿・出席・代返でガクチカクレジット獲得 | ④ |

## 3. ターゲットユーザー

### プライマリユーザー：ぼっち大学生（コネなし・人脈ゼロ）
- 日本国内の大学に在籍する学部生・大学院生
- 先輩・友人のコネクションがなく、楽単情報や過去問を得る手段がない
- 空きコマに一緒に遊ぶ人がおらず、時間を持て余している
- スマートフォンを日常的に使用

### セカンダリユーザー：連携企業
- 大学生の空き時間にアプローチしたい企業（映画館・出版社・就活サービス等）
- 大学生の空きコマ統計データを活用したいマーケター
- 採用活動を効率化したい企業（マイナビ等）

## 4. ガクチカクレジット — 独自トークンエコノミー

CLaaSの核心は **「ガクチカクレジット（Gakuchika Credit）」** という独自トークンです。

大学生が「怠慢」を実践するほどクレジットが貯まり、そのクレジットでさらに楽をできる好循環を生み出します。

### クレジット獲得アクション

| 怠慢な行動 | 獲得クレジット | 種別 | 対応機能 |
|---|---|---|---|
| 楽単情報を投稿する | +5 | 基礎点 | ① |
| 過去問をアップロードする | +10 | 基礎点 | ① |
| ノート・レポートを共有する | +8 | 基礎点 | ① |
| 授業評価を書く | +3 | 基礎点 | ① |
| 「役に立った」評価を獲得する | +1 | 動的報酬 | ① |
| 授業に出席する | +2 | 記録報酬 | ① |
| 出席票を誰かに渡す（QRコード） | +3 | リアル連携 | ② |
| 教材を無料で後輩に譲る | +5 | コミュニティ | ② |
| 企業クーポンを使う | クーポン設定値 | 企業連携 | ⑤ |
| 空きコマ情報を企業に提供する | +10 | データ提供 | ⑤ |

### クレジット消費アクション

| 使い道 | 消費クレジット | 対応機能 |
|---|---|---|
| 2学期目以降のAI時間割生成 | 設定値 | ① |
| プレミアムコンテンツアクセス | 設定値 | ③ |
| 代返依頼の報酬 | 設定値（依頼者が設定） | ② |
| 教材のクレジット交換 | 出品者が設定 | ② |

### エコシステムの循環

```
怠慢な行動（①投稿・出席 / ②代返・教材譲渡 / ⑤クーポン使用）
        ↓
  ④ガクチカクレジット獲得
        ↓
  さらに楽をする（①AI時間割・③プレミアムコンテンツ）
        ↓
  また怠慢な行動をする（好循環）
```

## 5. 5大機能の概要

### ① 楽単時間割自動生成機能
| 機能 | 概要 |
|---|---|
| 楽単情報共有 | 授業の難易度・評価方法・コメントを大学別に共有 |
| 過去問・ノート・レポート共有 | 試験の過去問・ノート・レポートをPDF/画像でアップロード・ダウンロード |
| AIシラバス解析 | シラバスをAIが解析し、出席重要度・レポート負荷・テスト依存率・難易度スコア（1-10）を算出 |
| AI自動時間割生成 | 最小努力の時間割を3パターン以上自動生成 |
| 同授業出席者情報共有 | 同じ授業の出席者情報を共有し、コネクション形成を支援 |

### ② マッチング機能
| 機能 | 概要 |
|---|---|
| 教材フリマ（売買） | 使い終わった教科書を無料/クレジット/現金で出品・購入 |
| 代返（ギグワーク） | 代返依頼・受諾・Trust_Scoreによる信頼性管理 |
| コミュニティマッチング | 合コン・麻雀・カフェ巡り等の趣味コミュニティで空きコマが重複するユーザーと自動マッチング |
| リアルタイムチャット | マッチングしたユーザーとのリアルタイムチャット・グループチャット |

### ③ 空きコマ娯楽レコメンド機能
| 機能 | 概要 |
|---|---|
| コンテンツ自動提案 | 空きコマの残り時間に合わせた娯楽・就活・資格情報を自動提案 |
| 地図連携 | 近くの映画館・美術館・カフェを地図で表示 |
| 読書提案 | 空きコマで読み終わる漫画・書籍を提案（平均読了時間ベース） |
| 就活・資格情報レコメンド | 空きコマの時間帯・興味カテゴリに合わせた就活情報・資格情報を配信 |

### ④ ガクチカクレジット（トークンエコノミー）
| 機能 | 概要 |
|---|---|
| クレジット獲得 | 楽単情報・過去問・ノート投稿、出席・代返・教材譲渡でクレジット獲得 |
| 企業連携クレジット | 企業クーポン使用・空きコマ情報提供でクレジット獲得 |
| クレジット消費 | AI時間割生成・プレミアム機能・代返報酬・教材購入にクレジットを利用 |
| 取引履歴管理 | クレジット残高・取引履歴の確認、不正検出ロジック |

### ⑤ 企業連携機能
| 機能 | 概要 |
|---|---|
| クーポン発行 | 企業がクーポンを発行し、学生にクレジットを付与 |
| 空きコマ統計提供 | 大学・学部・曜日・時限別の空きコマ人数（匿名化・集計）を企業に提供 |
| 企業向けダッシュボード | クーポン使用状況・統計データの閲覧・CSV出力 |
| 広告・就活情報配信 | 空きコマの時間帯に合わせた広告・就活情報を学生に配信 |

## 6. ビジネスモデル

```
大学生（ユーザー）
    ↓ ①②③を活用して怠慢な大学生活
④ガクチカクレジット獲得・消費（好循環）

企業（Partner）
    ↓ ⑤クーポン発行・空きコマデータ活用
CLaaS収益 → サービス継続・機能拡充
```

## 7. 非機能要件サマリー

| 項目 | 要件 | 採用技術 |
|---|---|---|
| セキュリティ | パスワードハッシュ化、JWT認証、個人情報暗号化 | Amazon Cognito、AWS KMS |
| プライバシー | 個人情報保護法準拠、オプトイン方式、k-匿名性（5名未満は非提供） | Amazon Macie |
| 不正防止 | Trust_Score、重複投稿検出、レート制限 | ElastiCache for Valkey、API Gateway |
| スケーラビリティ | サーバーレス・オートスケール | Lambda、Aurora Serverless v2 |
| 可観測性 | 分散トレーシング、パフォーマンス監視 | CloudWatch、AWS X-Ray |
| CI/CD | インフラのコード化、自動デプロイ | AWS CDK、CodePipeline |
