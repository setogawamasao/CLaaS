# 要件分析

## 1. コンセプト：「人（大学生）をダメにするサービス」

> Larry Wall はプログラマーの三大美徳の1つとして **「怠慢（Laziness）」** を挙げました。
> CLaaS はその精神を大学生活に適用します。

**CLaaS（Campus Life as a Service）** は、"大学生活をAIによって最適化しすぎた結果、人をダメにしてしまう"ことをコンセプトにした、生成AI時代の大学生向け生活最適化OSです。

従来、大学生活では「どの授業が楽か」「どの教授が厳しいか」「出席しなくてもレポートのみで単位が取得できるか」といった情報を、先輩とのつながりや経験、試行錯誤によって獲得してきました。

**しかしCLaaSでは、それらを生成AIが解析・最適化し、ユーザーへ提供します。**

ユーザーが履修情報やシラバスを登録すると、AIが授業ごとの「出席重要度」「レポート負荷」「テスト依存率」などを分析し、**最小努力で最大単位を取得できる履修**を提案します。

楽単情報によって生まれた空きコマには、「上映開始直前の映画」「同じ空きコマの学生による麻雀募集」「この時間で読み終わる漫画や書籍」をレコメンド。単位取得に必要な最低限の努力だけを残し、それ以外の時間を **"最も快適な暇つぶし"** へ最適化します。

**「大学生活を、クラウド化する。」** それが、Campus Life as a Service です。

## 2. 課題分析

### 大学生が抱える「面倒くさい」

#### 📚 学習面
| 面倒くさいこと | CLaaSの怠慢化 |
|---|---|
| 楽単情報を先輩から聞くのが面倒 | アプリで大学別に共有・検索 |
| シラバスを読んで難易度を判断するのが面倒 | AIが「出席重要度」「レポート負荷」「テスト依存率」を自動分析 |
| 時間割を自分で最適化するのが面倒 | AIが最小努力の時間割を3パターン自動生成 |
| 過去問を探すのが面倒 | アプリで大学・授業別に共有・ダウンロード |
| 使い終わった教科書を処分するのが面倒 | マーケットプレイスで後輩に譲渡 |

#### ⏰ 時間面
| 面倒くさいこと | CLaaSの怠慢化 |
|---|---|
| 空きコマに何をすればいいか考えるのが面倒 | 残り時間に合わせたコンテンツを自動提案 |
| 近くの映画館・施設を調べるのが面倒 | 地図で自動表示 |
| 読み終わらない本を選んでしまう | 空きコマで読み終わる作品だけを提案 |

#### 👥 コミュニティ面
| 面倒くさいこと | CLaaSの怠慢化 |
|---|---|
| 同じ空きコマの人を探すのが面倒 | 自動マッチング |
| 情報を共有する動機がない | ガクチカクレジットで報酬付与 |
| 授業に出席するのが面倒 | 代返機能（Trust_Scoreで信頼性管理） |

## 3. ターゲットユーザー

### プライマリユーザー：大学生（怠慢を極めたい人）
- 日本国内の大学に在籍する学部生・大学院生
- 「楽して単位を取りたい」「空きコマを有意義に過ごしたい」
- スマートフォンを日常的に使用

### セカンダリユーザー：連携企業
- 大学生の空き時間にアプローチしたい企業
- 映画館・出版社・就活サービス・スポーツ施設等
- 大学生の空きコマ統計データを活用したいマーケター

## 4. ガクチカクレジット — 独自トークンエコノミー

CLaaSの核心は **「ガクチカクレジット（Gakuchika Credit）」** という独自トークンです。

大学生が「怠慢」を実践するほどクレジットが貯まり、そのクレジットでさらに楽をできる好循環を生み出します。

### クレジット獲得アクション

| 怠慢な行動 | 獲得クレジット | 種別 |
|---|---|---|
| 楽単情報を投稿する | +5 | 基礎点 |
| 過去問をアップロードする | +10 | 基礎点 |
| ノート・レポートを共有する | +8 | 基礎点 |
| 授業評価を書く | +3 | 基礎点 |
| 「役に立った」評価を獲得する | +1 | 動的報酬 |
| 授業に出席する | +2 | 記録報酬 |
| 出席票を誰かに渡す（QRコード） | +3 | リアル連携 |
| 企業クーポンを使う | クーポン設定値 | 企業連携 |
| 空きコマ情報を企業に提供する | +10 | データ提供 |
| 教材を無料で後輩に譲る | +5 | コミュニティ |

### クレジット消費アクション

| 使い道 | 消費クレジット |
|---|---|
| 2学期目以降のAI時間割生成 | 設定値 |
| プレミアムコンテンツアクセス | 設定値 |
| 代返依頼の報酬 | 設定値（依頼者が設定） |
| 教材のトークン交換 | 出品者が設定 |

### エコシステムの循環

```
怠慢な行動（投稿・出席・代返）
        ↓
  ガクチカクレジット獲得
        ↓
  さらに楽をする（AI時間割・プレミアム機能）
        ↓
  また怠慢な行動をする（好循環）
```

## 5. 主要機能の概要

### 機能グループ1: 最小努力の履修計画
| 機能 | 概要 |
|---|---|
| 楽単情報共有 | 授業の難易度・評価方法・コメントを大学別に共有 |
| 過去問共有 | 試験の過去問をPDF/画像でアップロード・ダウンロード |
| AIシラバス解析 | シラバスをAIが解析し、難易度スコア（1-10）を算出 |
| AI自動時間割生成 | 最小努力の時間割を3パターン以上自動生成 |

### 機能グループ2: 空きコマの完全最適化
| 機能 | 概要 |
|---|---|
| コンテンツ提案 | 残り時間に合わせた娯楽・就活・資格情報を自動提案 |
| 地図連携 | 近くの映画館・美術館・カフェを地図で表示 |
| 読書提案 | 空きコマで読み終わる漫画・書籍を提案 |

### 機能グループ3: 怠慢を支え合うコミュニティ
| 機能 | 概要 |
|---|---|
| 空きコママッチング | 同じ空きコマのユーザーと自動マッチング |
| リアルタイムチャット | マッチングしたユーザーとチャット |
| コミュニティ | 麻雀・ゲーム・合コン等の趣味コミュニティ |
| 代返機能 | 代返依頼・受諾・Trust_Scoreによる信頼性管理 |

### 機能グループ4: 教材の循環
| 機能 | 概要 |
|---|---|
| 教材マーケットプレイス | 使い終わった教科書を無料/クレジット/現金で譲渡 |
| 評価システム | 取引後の相互評価でMarketplace_Trust_Scoreを管理 |

### 機能グループ5: 企業との共生
| 機能 | 概要 |
|---|---|
| クーポン発行 | 企業がクーポンを発行し、学生にクレジットを付与 |
| 空きコマ統計 | 大学・学部・曜日・時限別の空きコマ人数（匿名化・集計） |
| 企業ダッシュボード | クーポン使用状況・統計データの閲覧・CSV出力 |

## 6. ビジネスモデル

```
大学生（ユーザー）
    ↓ 怠慢な行動（投稿・出席・代返）
ガクチカクレジット獲得
    ↓ クレジット消費
さらに楽をする（好循環）

企業（Partner）
    ↓ クーポン発行・空きコマデータ活用
CLaaS収益
    ↓
サービス継続・機能拡充
```

## 7. 非機能要件サマリー

| 項目 | 要件 |
|---|---|
| セキュリティ | bcryptパスワードハッシュ化、JWT認証、個人情報暗号化 |
| プライバシー | 個人情報保護法準拠、オプトイン方式、k-匿名性（5名未満は非提供） |
| 不正防止 | Trust_Score、重複投稿検出、レート制限 |
| スケーラビリティ | マイクロサービス構成（17サービス）による独立スケール |
