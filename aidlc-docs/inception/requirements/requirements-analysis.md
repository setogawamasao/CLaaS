# 要件分析

## 1. プロジェクト概要

### アプリ名
**CLaaS（Campus Life as a Service）**

### コンセプト
大学生活に関わるあらゆる「困った」と「もったいない」を解決するオールインワンプラットフォーム。情報共有・AI活用・コミュニティ形成・企業連携を一つのエコシステムで実現する。

---

## 2. 課題分析

### 大学生が抱える課題

#### 📚 学習面
| 課題 | 詳細 |
|---|---|
| 履修選択の情報不足 | どの授業が楽か、過去問はどこにあるか、先輩から聞くしかない |
| 時間割作成の手間 | 単位数・難易度・時間帯を考慮した最適な時間割を自力で作るのは大変 |
| 教材コストの高さ | 1冊数千円の教科書を毎学期購入、使い終わると捨てるしかない |

#### ⏰ 時間面
| 課題 | 詳細 |
|---|---|
| 空きコマの無駄遣い | 1〜2時間の空きコマをどう使えばいいかわからない |
| 就活準備の後回し | 空きコマがあっても何をすればいいかわからず、スマホを見て終わる |

#### 👥 コミュニティ面
| 課題 | 詳細 |
|---|---|
| 同じ空きコマの人を探せない | 一緒に時間を過ごしたい人がいても見つける手段がない |
| 情報共有への動機がない | 楽単情報を知っていても、わざわざ共有する理由がない |

---

## 3. ターゲットユーザー

### プライマリユーザー：大学生
- 日本国内の大学に在籍する学部生・大学院生
- スマートフォンを日常的に使用
- 履修計画・就活・大学生活の充実に関心がある

### セカンダリユーザー：連携企業
- 大学生をターゲットにしたマーケティングを行いたい企業
- 映画館・出版社・就活サービス・スポーツ施設等
- 大学生の空き時間データを活用したい企業

---

## 4. 主要機能の概要

### 機能グループ1: 学習支援
| 機能 | 概要 |
|---|---|
| 楽単情報共有 | 授業の難易度・評価方法・コメントを大学別に共有 |
| 過去問共有 | 試験の過去問をPDF/画像でアップロード・ダウンロード |
| AIシラバス解析 | シラバスをAIが解析し、難易度スコア（1-10）を算出 |
| AI自動時間割生成 | 大学・学部・年度を入力するだけで最適な時間割を3パターン生成 |

### 機能グループ2: 空きコマ活用
| 機能 | 概要 |
|---|---|
| コンテンツ提案 | 空きコマの残り時間に合わせた娯楽・就活・資格情報を提案 |
| 地図連携 | 近くの映画館・美術館・カフェを地図で表示 |
| 読書提案 | 空きコマで読み終わる漫画・書籍を提案 |

### 機能グループ3: コミュニティ
| 機能 | 概要 |
|---|---|
| 空きコママッチング | 同じ空きコマのユーザーとマッチング |
| リアルタイムチャット | マッチングしたユーザーとチャット |
| コミュニティ | スポーツ・ゲーム・合コン等の趣味コミュニティ |
| 代返機能 | 代返依頼・受諾・Trust_Scoreによる信頼性管理 |

### 機能グループ4: マーケットプレイス
| 機能 | 概要 |
|---|---|
| 教材譲渡 | 使い終わった教科書・テキストを無料/クレジット/現金で譲渡 |
| 評価システム | 取引後の相互評価でMarketplace_Trust_Scoreを管理 |

### 機能グループ5: ガクチカクレジットエコノミー
| 行動 | 獲得クレジット |
|---|---|
| 楽単情報投稿 | +5クレジット（基礎点） |
| 過去問アップロード | +10クレジット（基礎点） |
| ノート・レポート提供 | +8クレジット（基礎点） |
| 授業評価投稿 | +3クレジット（基礎点） |
| 「役に立った」評価獲得 | +1クレジット（動的報酬） |
| 出席記録 | +2クレジット |
| 出席票提供 | +3クレジット |
| 企業クーポン使用 | クーポン設定値 |
| 空きコマ情報提供同意 | +10クレジット |
| 無料教材譲渡 | +5クレジット |

### 機能グループ6: 企業向け機能
| 機能 | 概要 |
|---|---|
| クーポン発行 | 企業がクーポンを発行し、ユーザーに配信 |
| 空きコマ統計 | 大学・学部・曜日・時限別の空きコマ人数（匿名化・集計） |
| 企業ダッシュボード | クーポン使用状況・統計データの閲覧・CSV出力 |

---

## 5. ビジネスモデル

```
ユーザー（大学生）
    ↓ 情報提供・行動
ガクチカクレジット獲得
    ↓ クレジット消費
プレミアム機能利用（2学期目以降）

企業（Partner）
    ↓ クーポン発行・空きコマデータ購入
CLaaS収益
    ↓
サービス継続・機能拡充
```

## 6. 非機能要件サマリー

| 項目 | 要件 |
|---|---|
| セキュリティ | bcryptパスワードハッシュ化、JWT認証、個人情報暗号化 |
| プライバシー | 個人情報保護法準拠、オプトイン方式、k-匿名性（5名未満は非提供） |
| 可用性 | 主要機能の24時間稼働 |
| スケーラビリティ | マイクロサービス構成（17サービス）による独立スケール |
| 不正防止 | Trust_Score、重複投稿検出、レート制限 |
