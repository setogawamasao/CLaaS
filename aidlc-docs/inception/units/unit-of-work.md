# Unit of Work 計画

## 概要

CLaaSの実装を段階的に進めるための作業単位（Unit of Work）の計画です。

**コンセプト「人をダメにする」を実現するため、以下の優先順位で実装します：**
1. まず「楽をする仕組み」（楽単・AI時間割）を作る
2. 次に「楽をした報酬」（ガクチカクレジット）を作る
3. そして「楽した時間の使い道」（コミュニティ・マーケット）を作る
4. 最後に「企業との共生」（クーポン・統計）を作る

依存関係を考慮し、基盤から順番に実装します。

---

## 実装フェーズ全体像

```
Phase 1: 基盤構築
    ↓
Phase 2: コア機能（学習支援）
    ↓
Phase 3: ガクチカクレジットエコノミー
    ↓
Phase 4: コミュニティ・マッチング
    ↓
Phase 5: マーケットプレイス
    ↓
Phase 6: 企業向け機能
    ↓
Phase 7: AI高度化・最適化
```
