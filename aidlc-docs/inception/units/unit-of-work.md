# Unit of Work 計画

## 概要

CLaaSの実装を段階的に進めるための作業単位（Unit of Work）の計画です。

**5大機能を以下の順序で実装します：**
1. まず基盤を構築する
2. **①楽単時間割自動生成機能**（コア価値）を実装する
3. **④ガクチカクレジット**（エコノミーの基盤）を実装する
4. **②マッチング機能**（コミュニティ・代返・教材フリマ）を実装する
5. **③空きコマ娯楽レコメンド機能**を実装する
6. **⑤企業連携機能**を実装する
7. AI高度化・最適化を行う

---

## 実装フェーズ全体像

```
Phase 1: 基盤構築
    ↓
Phase 2: ①楽単時間割自動生成機能
    ↓
Phase 3: ④ガクチカクレジット（トークンエコノミー）
    ↓ 並行可能
Phase 4: ②マッチング機能    Phase 5: ③空きコマ娯楽レコメンド    Phase 6: ⑤企業連携機能
    ↓ すべて完了後
Phase 7: AI高度化・最適化
```

- [ ] **Amazon S3** バケット + **CloudFront** ディストリビューション作成
- [ ] **Amazon Cognito** ユーザープール作成（大学メール検証）
- [ ] 環境変数管理（**AWS Secrets Manager**）

### Unit 1.2: データベース基盤
- [ ] **Prisma** ORM導入（Aurora Serverless v2接続）
- [ ] users テーブル作成
- [ ] universities / faculties / departments テーブル作成
- [ ] courses テーブル作成
- [ ] 日本国内大学データの初期投入
- [ ] **Amazon OpenSearch Serverless** インデックス設計

### Unit 1.3: 認証サービス（Amazon Cognito）
- [ ] Cognito ユーザープール設定（大学メール検証）
- [ ] Hono + Cognito JWT検証ミドルウェア

### Unit 2.3: 過去問・ノート共有サービス（Past Exam Service）

- [ ] AWS Textract + Macie連携（PDF解析・ウイルス検査）
